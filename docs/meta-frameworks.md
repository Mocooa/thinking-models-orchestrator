# Meta Framework（元框架）
**组合思维模型的高级玩法**

> 单一模型解决简单问题，组合模型解决复杂问题。Meta Framework 是在一次分析中**动态调用多个思维模型**的系统性方法。

---

## 快速导航

| 你的情况 | 推荐 Framework | 5 分钟简化版 |
|----------|----------------|-------------|
| 需要创新想法 | Diverge-Converge | SCAMPER → Pros/Cons → 选最佳 |
| 需要完整方案 | CGEC | 5 Whys → ToT → Inversion → Minto |
| 持续迭代项目 | PAR | Plan → Do → Review → Repeat |
| 进入新市场 | Outside-In | 环境 → 行业 → 竞争 → 客户 → 行动 |
| 重大决策风险高 | Devil's Advocate | 支持方 vs 反对方 → 综合 |
| 时间紧、变化快 | OODA Loop | Observe → Orient → Decide → Act |
| 用户中心设计 | Double Diamond | 发现 → 定义 → 开发 → 交付 |
| 全面多角度思考 | Six Thinking Hats | 6 顶帽子轮流戴 |
| 不知道用什么方法 | Cynefin | 先分类问题复杂度 |
| 目标明确路径不清 | Working Backwards | 从终点倒推到今天 |
| 问题反复出现 | Iceberg Model | 事件 → 模式 → 结构 → 心智 |

---

## 为什么需要 Meta Framework？

| 问题类型 | 单一模型 | 组合模型 |
|----------|---------|---------|
| 简单事实查询 | 足够 | 过度 |
| 复杂决策 | 不足 | 必要 |
| 创新 + 评估 | 偏科 | 互补 |
| 多利益相关方 | 单视角 | 多维度 |

> **核心洞察**：每个模型都有盲区，组合使用可以**互相校正**。

---

## Framework 1: Diverge-Converge（发散-收敛）

**模式**：先大量产生选项，再系统筛选

**发散阶段** → SCAMPER + Lateral Thinking + First Principles（产出 20+ 想法）
**收敛阶段** → Quick Filter → Weighted Decision Matrix → Second-Order + Inversion

**适用场景**：产品创新、战略规划、解决开放性问题

**5 分钟简化版**：
1. SCAMPER 快速出 14 个想法（每维度 2 个）
2. 快速排除明显不可行的
3. 对 Top 3 做 Pros/Cons
4. 选最佳，定 Backup

---

## Framework 2: CGEC (Clarify-Generate-Evaluate-Communicate)

**模式**：完整问题解决闭环

| 阶段 | 目标 | 常用模型 |
|------|------|---------|
| **C** larify | 澄清问题本质 | 5 Whys, MECE |
| **G** enerate | 生成多条路径 | ToT, SCAMPER |
| **E** valuate | 评估风险与影响 | Inversion, Second-Order, Pros/Cons |
| **C** ommunicate | 结构化表达 | Golden Circle, Minto Pyramid |

**适用场景**：咨询项目、商业提案、需要向高管汇报的决策

**5 分钟简化版**：
- C: 5 Whys 找根因
- G: ToT 出 3-5 条路径
- E: Inversion 检验风险
- C: Minto Pyramid 汇报

---

## Framework 3: PAR (Plan-Act-Reflect)

**模式**：行动 + 反思循环

- **Plan**：ToT 规划路径
- **Act**：ReAct 执行验证
- **Reflect**：事后复盘（预期 vs 实际）

**适用场景**：迭代式项目、学习新技能、持续改进流程

---

## Framework 4: Outside-In Analysis

**模式**：从外部环境逐层聚焦到具体行动

宏观环境 → 行业分析 → 竞争对手 → 客户需求 → 我的定位 → 具体行动

**适用场景**：市场进入策略、产品定位、竞争分析

**5 分钟简化版**：
1. PESTLE 扫一遍宏观
2. Porter's 5 Forces 看行业
3. 竞品对比表
4. JTBD 看客户
5. Golden Circle 定位自己

---

## Framework 5: Devil's Advocate（对抗性思考）

**模式**：强制引入反对意见

初始方案 → Advocate 支持论证 + Devil 攻击论证 → 综合判断 → 加强版方案

**适用场景**：重大投资决策、风险管理、避免确认偏误

---

## Framework 6: OODA Loop（快速决策循环）

来源：美国空军战斗机飞行员 John Boyd

**Observe**（观察）→ **Orient**（定向）→ **Decide**（决策）→ **Act**（行动）→ 循环

**核心洞察**：Orient（定向）是关键——你的心智模型决定你如何解读信息。

**适用场景**：市场突变、竞争对手出招、危机管理、快速试错

---

## Framework 7: Double Diamond（双钻设计思维）

来源：英国设计协会

**钻石 1：找对问题** → Discover（发现）→ Define（定义）
**钻石 2：找对方案** → Develop（开发）→ Deliver（交付）

**核心洞察**：先发散再收敛，做两次——第一次确保解决正确的问题，第二次确保用正确的方案。

**适用场景**：产品功能设计、服务体验优化、内容选题策划

---

## Framework 8: Six Thinking Hats（六顶思考帽）

来源：Edward de Bono

| 帽子 | 视角 | 问什么 |
|------|------|--------|
| 白帽 | 事实 | 我们知道什么？数据说什么？ |
| 红帽 | 情感 | 我的直觉/感受是什么？ |
| 黑帽 | 风险 | 什么可能出错？ |
| 黄帽 | 价值 | 好处是什么？为什么能成？ |
| 绿帽 | 创意 | 还有什么可能？ |
| 蓝帽 | 流程 | 我们的思考过程对吗？ |

**适用场景**：团队决策、评估新机会、冲突调解

---

## Framework 9: Cynefin（问题复杂度判断框架）

来源：Dave Snowden

| 领域 | 特征 | 策略 | 示例 |
|------|------|------|------|
| **Simple** | 因果明确 | 套用最佳实践 | 报销流程 |
| **Complicated** | 因果可分析 | 找专家分析 | 系统架构 |
| **Complex** | 因果只能事后看清 | 小步试错 | 创业、市场预测 |
| **Chaotic** | 无因果可循 | 先稳定局面 | 危机公关 |

**核心洞察**：用错误的方法解决问题比不解决更糟。

---

## Framework 10: Working Backwards（逆向工作法）

来源：Amazon

从理想终态倒推到现在：
理想终态 → 倒数第 2 步 → 倒数第 3 步 → ... → 现在该做什么

**核心洞察**：从"未来完成时"倒推，比从现在往前想更清晰。

**适用场景**：产品规划、个人目标设定、项目里程碑规划

---

## Framework 11: Iceberg Model（冰山模型）

来源：系统思考 (Systems Thinking)

| 层级 | 问什么 | 干预效果 |
|------|--------|----------|
| Events 事件 | 发生了什么？ | 救火（治标）|
| Patterns 模式 | 这种情况多久发生一次？| 预测（适应）|
| Structures 结构 | 什么机制导致这个模式？| 改变规则（治本）|
| Mental Models 心智 | 什么信念维持这个结构？| 转型（根治）|

**适用场景**：问题反复出现、组织变革、个人习惯改变

## 自定义 Meta Framework

### 模块化组合公式

1. **定义问题类型**：开放性 / 分析性 / 决策性 / 沟通性
2. **选择各阶段模型**：
   - 澄清阶段（选 1-2）：5 Whys, MECE, Socratic
   - 生成阶段（选 1-3）：SCAMPER, First Principles, Lateral Thinking, ToT, CoT
   - 评估阶段（选 1-2）：Pros/Cons, Second-Order, Inversion, Pre-mortem
   - 表达阶段（选 1）：Golden Circle, Minto Pyramid
3. **组装执行**

---

## 使用建议

1. **不要过度工程化**：简单问题用单一模型
2. **灵活跳过阶段**：如果问题已经很清晰，跳过 Clarify
3. **迭代而非一次性**：复杂问题需要多轮 Framework
4. **输出很重要**：COMMUNICATE 阶段不能省略
5. **建立个人偏好**：发现适合自己的组合后固化下来
